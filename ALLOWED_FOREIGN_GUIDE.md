# Руководство по управлению словарем разрешенных иностранных терминов

## Назначение

Словарь `allowed_foreign.json` содержит список иностранных слов, которые **разрешены к использованию** без ограничений согласно закону № 168-ФЗ. Это позволяет системе не отмечать как нарушения:

- **Международные стандарты** (размеры одежды: XS, S, M, L, XL)
- **Собственные имена и названия** (модели, коллекции: Row, Model, Collection)
- **Устоявшиеся профессиональные термины** (материалы, стили, сезоны)
- **Термины без русских аналогов** (технические обозначения)

## Структура файла

```json
{
  "name": "allowed_foreign",
  "version": "1.0",
  "source": "демо-словарь",
  "description": "Описание словаря",
  "words": ["xs", "s", "m", "l", "xl", ...]
}
```

## Как добавлять новые термины

1. Откройте файл `dictionaries/data/allowed_foreign.json`
2. Добавьте новые термины в массив `words` в нижнем регистре
3. Сохраните файл
4. Перезапустите сервис: `docker-compose restart checker`

## Пример добавления термина

```json
{
  "words": [
    "xs",
    "s",
    "m",
    "l",
    "xl",
    "ваш_новый_термин"
  ]
}
```

## Правовые основания

Согласно закону № 168-ФЗ "О защите русского языка":

- **Статья 5** требует приоритета русского языка, но допускает использование иностранных терминов, если:
  - Они являются международными стандартами
  - Отсутствует русский аналог
  - Это устоявшиеся профессиональные термины

- **Статья 6** защищает чистоту языка от искажений, но не запрещает использование общепринятых терминов

- **Статья 6.1** запрещает только ненормативную лексику

Словарь `allowed_foreign` помогает системе корректно применять эти нормы.

## Актуальность словаря

Текущий демо-словарь содержит **58 терминов**, охватывающих основные категории fashion-индустрии.

### Как поддерживать актуальность:

1. **Регулярный обзор** - анализировать проверенные тексты на предмет новых терминов
2. **Мониторинг отрасли** - отслеживать появление новых международных стандартов
3. **Обратная связь** - добавлять термины, которые пользователи считают разрешенными
4. **Юридическая экспертиза** - консультироваться с юристами по языковому праву

### Источники для пополнения:

- ГОСТы и международные стандарты (ISO, EN)
- Профессиональная терминология (модная индустрия, дизайн)
- Устоявшиеся заимствования, вошедшие в русский язык
- Рекомендации Роскомнадзора и Минюста

## Верификация терминов

Перед добавлением термина задайте вопросы:

1. ✅ Это международный стандарт? (размеры, единицы измерения)
2. ✅ Это собственное имя/бренд/модель? (не требует перевода)
3. ✅ Нет русского аналога? (технические термины)
4. ✅ Это устоявшийся профессиональный жаргон? (принят в отрасли)
5. ❌ Это просто маркетинговый прием? (лучше заменить на русский)

## Примеры

### Разрешить:
- `xs`, `s`, `m`, `l`, `xl` - размеры
- `row` - название коллекции/модели
- `winter` - сезон (в названии "Winter Collection")
- `cotton` - материал (устоявшийся термин)

### Не разрешать (есть аналоги):
- `new` → "новый", "новинка"
- `sale` → "распродажа"
- `meeting` → "встреча", "совещание"
- `manager` → "менеджер", "руководитель"

## Технические детали

- Словарь загружается автоматически при старте сервиса
- Все слова хранятся в нижнем регистре
- Проверка case-insensitive
- Изменения вступают в силу после перезапуска сервиса

## Использование через API

### Базовый пример
```bash
curl -X POST "http://localhost:8000/api/v1/check" \
  -H "Content-Type: application/json" \
  -d '{"text": "New collection available"}'
```
Результат: слово "New" будет помечено как иностранное (если нет в allowed_foreign.json)

### С передачей дополнительных разрешенных слов
```bash
curl -X POST "http://localhost:8000/api/v1/check" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "New collection available",
    "allowed_words": ["new", "collection"]
  }'
```
Результат: слова "New" и "collection" будут иметь статус "allowed" и не считаться нарушениями.

### С URL
```bash
curl -X POST "http://localhost:8000/api/v1/check" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/article",
    "allowed_words": ["premium", "exclusive"]
  }'
```

### Приоритет разрешенных слов
1. Слова из `allowed_foreign.json` (загружаются при старте)
2. Слова из параметра `allowed_words` (добавляются динамически)
3. Остальные иностранные слова проверяются по обычным правилам

### Примеры на Python
```python
import requests

# Проверка с дополнительными разрешенными словами
response = requests.post(
    "http://localhost:8000/api/v1/check",
    json={
        "text": "New winter collection with XS sizes",
        "allowed_words": ["premium", "exclusive"]  # доп. разрешенные
    }
)
result = response.json()
```

## Связь с другими словарями

- **`иностранные_слова`** - базовый список иностранных слов, требующих внимания
- **`русские_аналоги`** - словарь соответствия иностранных слов и их русских вариантов
- **`allowed_foreign`** - исключения: термины, которые можно использовать без ограничений

Приоритет проверки:
1. Запрещенные слова (высший приоритет)
2. Разрешенные иностранные термины (allowed_foreign)
3. Иностранные слова с русскими аналогами (русские_аналоги)
4. Другие иностранные слова (иностранные_слова)
5. Русские слова (ru_words_github, hunspell_ru)
