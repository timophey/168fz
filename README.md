# Проверка текста на соответствие закону № 168-ФЗ

Мини-приложение на Python для проверки текстов на соответствие Федеральному закону № 168-ФЗ "О защите русского языка как государственного языка Российской Федерации".

## ⚠️  Важное замечание о словарях

Приложение поддерживает два режима работы со словарями:

1. **Демо-словари** (включены по умолчанию) - содержат ограниченный набор слов для тестирования
2. **Официальные словари** - требуют синхронизации с источниками, указанными в законе

Для production использования необходимо настроить синхронизацию с официальными словарями.

## 🌐 Веб-интерфейс

Приложение включает полноценный веб-интерфейс с возможностью:

- ✅ **Проверки текста** в реальном времени
- ✅ **Проверки по URL** - автоматическое извлечение текста с веб-страниц
- ✅ **Проверки файлов** - загрузка TXT, MD, HTML, JSON, CSV
- 📊 **Мониторинга** загруженных словарей (публичный)
- 🔄 **Синхронизации** словарей (требует админ-прав)
- 📥 **Загрузки** пользовательских словарей (требует админ-прав)
- 📥 **Скачивания** отчетов в различных форматах

Откройте http://localhost:8000 после запуска.

### Публичная и админ-панель

Приложение разделено на две части:

- **Публичная часть** (`/`) - доступна всем. Включает проверку текста, просмотр статуса словарей и управление разрешенными иностранными словами.
- **Админ-панель** (`/admin`) - требует админ-ключа. Включает синхронизацию словарей и управление пользовательскими словарями.


## 🔐 Админ-доступ

Для доступа к админ-панели и операциям управления словарями требуется админ-ключ.

### Настройка админ-ключа

1. Установите переменную окружения `ADMIN_KEY` в файле `.env`:
   ```env
   ADMIN_KEY=ваш_секретный_ключ
   ```

2. Перезапустите приложение.

3. При переходе в админ-панель (`/admin`) система запросит ключ. Ключ можно передать:
   - Через URL: `/admin?admin_key=ваш_ключ`
   - В заголовке `X-Admin-Key` (для API запросов)

### Защищенные API endpoints

Следующие endpoints требуют админ-ключа:

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `POST` | `/api/v1/dictionaries/load` | Загрузка словаря (устарел, используйте import) |
| `POST` | `/api/v1/dictionaries/import` | Импорт словаря из XLSX |
| `GET` | `/api/v1/dictionaries/{name}/export` | Экспорт словаря в XLSX |
| `DELETE` | `/api/v1/dictionaries/{name}` | Удаление пользовательского словаря |
| `GET` | `/api/v1/dictionaries/template/xlsx` | Скачивание шаблона XLSX |
| `POST` | `/api/v1/sync/{dict_name}` | Синхронизация конкретного словаря |
| `POST` | `/api/v1/sync/all` | Синхронизация всех словарей |

Пример использования с админ-ключом:
```bash
curl -X POST "http://localhost:8000/api/v1/sync/all" \
  -H "X-Admin-Key: ваш_ключ"
```

## 📦 Пользовательские словари

Приложение поддерживает пользовательские словари, которые хранятся отдельно от официальных и не синхронизируются с внешними источниками.

### Хранение

- **Официальные словари**: `dictionaries/data/` (синхронизируются, могут быть в git)
- **Пользовательские словари**: `dictionaries/user_data/` (НЕ в git, должны быть в Docker volume)

В Docker Compose volume `./dictionaries/user_data:/app/dictionaries/user_data` монтируется автоматически.

### Формат XLSX

Пользовательские словари работают через формат XLSX со следующей структурой:

| Колонка | Обязательная | Описание |
|---------|--------------|----------|
| `word` | ✅ | Слово или фраза |
| `russian_analog` | ❌ | Русский аналог (для словарей типа "русские аналоги") |
| `category` | ❌ | Категория словаря |
| `description` | ❌ | Описание словаря |

### Работа с пользовательскими словарями

1. **Скачайте шаблон** - в админ-панели нажмите "Скачать шаблон XLSX"
2. **Заполните файл** - добавьте слова в колонку `word`, при необходимости заполните другие колонки
3. **Импортируйте** - в админ-панели загрузите XLSX файл, укажите имя, категорию и описание (можно перезаписать значения из файла)
4. **Управление** - в таблице "Загруженные словари" можно:
   - **Экспортировать** - скачать словарь в XLSX для редактирования
   - **Удалить** - удалить пользовательский словарь

### Важно

- Пользовательские словари имеют префикс `user_` в имени (если не указан явно)
- При перезаписи существующего словаря версия автоматически инкрементируется
- Категория выбирается из списка: Запрещенные слова, Иностранные слова, Разрешенные иностранные термины, Нормативные слова, Термины, Аббревиатуры, Топонимы, Профессионализмы и жаргон, Другие словари
- Поле `russian_analog` создает маппинг {слово: русский_аналог} для улучшения рекомендаций
- Максимальный размер файла: 10MB

## 📚 Функциональность

Приложение анализирует текст и определяет:

- **Запрещенные слова** (ненормативная лексика) - требующие немедленного удаления (ст. 6.1)
- **Иностранные слова** - классифицируются по трём статусам:
  - `allowed` - разрешённые иностранные термины (международные стандарты, собственные имена)
  - `foreign` - иностранные слова без установленного русского аналога (рекомендуется заменить)
  - `foreign_with_alternative` - иностранные слова, имеющие утверждённый русский аналог (нарушение ст. 6)
- **Нарушения норм** (ст. 6) - использование иностранного слова при наличии утверждённого русского аналога

Результат представляется в виде подробного отчета с указанием:
- Статистики по тексту (общее количество слов, уникальные слова)
- Использованных словарей и их категорий
- Полного списка всех слов с их статусами (`all_words`)
- Списка проблемных слов с указанием статьи закона
- Рекомендаций по исправлению
- Уровня риска (`low`, `medium`, `high`)

### Недавние улучшения (2026-03-26/27)

- ✅ **Согласованность данных**: `all_words` и `checks` теперь формируются за один проход, что обеспечивает полную согласованность.
- ✅ **Улучшенная классификация**: добавлены новые словари (сокращения, технические термины, профессионализмы, топонимы) и система категоризации.
- ✅ **Корректный уровень риска**: пересмотрена логика оценки риска. Уровень `medium` присваивается, если доля иностранных слов >10% от текста или их количество >=3, а также при наличии >5 нарушений. Запрещённые слова дают `high`. Это снижает ложные срабатывания на тексты с единичными иностранными словами.
- ✅ **Настраиваемые пороги риска**: добавлены переменные окружения `RISK_THRESHOLD_VIOLATIONS`, `RISK_THRESHOLD_FOREIGN_RATIO`, `RISK_THRESHOLD_FOREIGN_COUNT` для тонкой настройки чувствительности оценки риска.
- ✅ **Улучшенные рекомендации**: словарь `русские_аналоги` теперь содержит конкретные маппинги иностранных слов и их русских эквивалентов, что позволяет давать более точные предложения по замене.
- ✅ **Расширенные тесты**: добавлен комплексный набор тестов на крайние случаи (смешанные языки, регистр, пунктуация, вариации слов) и тесты для настраиваемых порогов.
- ✅ **Документация**: обновлено описание классификации, конфигурации и функциональности.

## 🔄 Синхронизация словарей

### Доступные источники

**Реальные (автоматическая загрузка):**
- `нормативный_словарь` - 1.5M+ русских слов (полная версия)
- `hunspell_ru` - Hunspell словарь (LibreOffice) ~200K слов
- `opencorpora` - корпус русского языка ~500K словоформ
- `thesaurus_github` - тезаурус с синонимами и антонимами

**Локальные (демо-словари, не синхронизируются):**
- `сокращения_аббревиатуры` - сокращения и аббревиатуры (IT, бизнес, гос.)
- `технические_термины` - технические термины (IT, программирование, сети)
- `профессионализмы_жаргон` - профессиональный жаргон и сленг
- `топонимы` - географические названия (страны, города, регионы)
- `иностранные_слова` - иностранные слова
- `запрещенные_слова` - запрещённые слова
- `русские_аналоги` - соответствие иностранных слов русским аналогам
- `allowed_foreign` - разрешённые иностранные термины

**Legacy (официальные, требуют ручной настройки):**
- `gramota_ru` - Грамота.ру
- `opencorpora_official` - официальный OpenCorpora

### Команды синхронизации

```bash
# Через веб-интерфейс: кнопка "Синхронизировать все словари" в админ-панели (/admin)

# Через API (требуется админ-ключ)
curl -X POST "http://localhost:8000/api/v1/sync/all" \
  -H "X-Admin-Key: ваш_ключ"
```

## 📊 Классификация слов

Приложение использует гибкую систему классификации словарей по категориям:

| Категория | Описание | Примеры словарей |
|-----------|----------|------------------|
| **Нормативные слова** | Стандартные русские слова | `нормативный_словарь`, `hunspell_ru`, `opencorpora` |
| **Иностранные слова** | Слова иностранного происхождения | `иностранные_слова`, `русские_аналоги` |
| **Запрещенные слова** | Ненормативная лексика | `запрещенные_слова` |
| **Разрешенные иностранные термины** | Иностранные термины, допустимые в русском языке | `allowed_foreign` |
| **Термины** | Специальные термины и понятия | `thesaurus_github` |
| **Аббревиатуры** | Сокращения и аббревиатуры | `сокращения_аббревиатуры` |
| **Технические термины** | IT и технические термины | `технические_термины` |
| **Профессионализмы и жаргон** | Профессиональный сленг | `профессионализмы_жаргон` |
| **Топонимы** | Географические названия | `топонимы` |

Категории можно настраивать через файл [`dictionaries/category_mapping.json`](dictionaries/category_mapping.json). Для словарей, у которых указано поле `category` в метаданных, этот приоритетнее маппинга.

## 🐛 Устранение неполадок

### Словари не загружены
- Проверьте, что volume `dictionaries/data` смонтирован
- Запустите синхронизацию через веб-интерфейс или CLI

### Ошибка при синхронизации
- Проверьте сетевой доступ из контейнера
- Некоторые источники могут быть недоступны
- Используйте fallback словари
