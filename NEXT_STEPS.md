- **Файлы**: `checker.py`, `app/main.py`
- **Изменения**:
  - Объединена логика проверки текста: `checker.check_text()` теперь формирует и `checks`, и `all_words` за один проход.

## Проблемы, требующие дальнейшего внимания

### 2. Нормативные нарушения не работают
- **Проблема**: `_check_normative_usage()` всегда возвращает `True`, поэтому `normative_violations` остаётся пустым.
- **Решение**:
  - Либо реализовать реальную проверку контекста (например, с помощью NLP или правил).
  - Либо убрать проверку нормативных нарушений, если она не требуется.
  - Если оставить, то нужно также увеличивать `summary.violation_count` за нормативные нарушения.

### 3. Уровень риска занижен
- **Проблема**: `_assess_risk()` определяет уровень `low` для большинства текстов, даже при наличии иностранных слов.
- **Решение**: Пересмотреть логику:
  - `high` — если есть запрещённые слова.
  - `medium` — если `violation_count > 5` или `len(foreign_words) > 3` (или другое разумное порог).
  - `low` — иначе.
  - Учитывать все типы нарушений (запрещённые, иностранные, нормативные).

### 4. Расширение словарей
- **Проблема**: Многие иностранные слова (бренды, новые термины) отсутствуют в словаре `иностранные_слова`, что могло бы снизить чувствительность в старом коде. Теперь это менее критично, но наличие в словаре улучшает рекомендации.
- **Решение**: Регулярно синхронизировать словари из внешних источников (используя `/api/v1/sync/all`). Проверить, что `ru_words_github` не содержит иностранных слов, которые должны быть помечены как `foreign`.

### 5. Производительность и масштабируемость
- **Улучшение**: Устранена двойная обработка текста (было: `checker.check_text()` + `extract_all_words_with_status()`). Теперь один проход.
- **Дальше**: При больших текстах можно рассмотреть асинхронную обработку или кэширование результатов.

## Тестирование после изменений

### Прямой вызов `checker.check_text()` (подтверждено)
```python
from checker import LanguageChecker
checker = LanguageChecker()
result = checker.check_text("Hairguru Cashback")
# result['checks']['foreign_words'] содержит оба слова
# result['all_words'] имеет статус 'foreign' для этих слов
```

### Запуск сервера
После внесения изменений в код необходимо перезапустить сервер:
```bash
# Если сервер запущен через uvicorn напрямую:
pkill -f "app.main"  # может потребовать sudo
python -m app.main

# Или через make:
make run
```

### Запуск тестов
```bash
python test_urls.py
```
Ожидаемый результат: в `test_results.json` для каждого URL `checks.foreign_words` должен содержать слова со статусом `foreign` из `all_words`, а `summary.has_foreign` должно быть `true` для сайтов с иностранными словами.

## Примечания
- В `checker.py` логика определения русских словарей исключает явно иностранные словари (`hunspell_en`, `foreign_words_github` и др.) и русские аналоги. Это обеспечивает корректную классификацию.
- Статус `foreign_with_alternative` присваивается только если слово есть в `русские_аналоги`.
- Нормативные нарушения пока не реализованы (требуют отдельной работы).
